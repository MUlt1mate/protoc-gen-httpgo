# Contributing to httpgo

First off, thank you for taking the time to contribute!

## How Can I Contribute?

### Tackle the TODO List

If you're looking for a place to start, check the TODO section in our README.md. These are items that I am
actively looking to implement.

### Reporting Bugs

Check the to see if the bug has already been reported.

If not, open a new issue. Include a clear title, a description of the problem, and steps to reproduce the error.

### Suggesting Enhancements

Open an issue to discuss the change you'd like to make.

## Development Workflow

- all tests must succeed
- there should be no linter errors
- if you want to create an exception for linter, please write a reason to do so in code comment 
- run `make all` and check the output before creating pull request to ensure that all checks passed
